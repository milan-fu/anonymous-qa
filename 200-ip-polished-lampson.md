# 匿名问答网站 — 架构与逻辑设计

## 一、项目概述

一个匿名问答网站，包含两个核心页面：
- **提问 & 浏览页**（公开）：用户可以匿名提问，并浏览已公开展示的问答。
- **管理回答页**（管理员）：网站主可以查看所有提问、回答问题、控制展示状态。

---

## 二、技术栈选择

| 层级 | 技术 | 理由 |
|------|------|------|
| 后端 | **Python + Flask** | 轻量、简单、无需构建步骤，适合小项目 |
| 数据库 | **SQLite** | 零配置，单文件，数据量不大时足够 |
| 前端 | **HTML + CSS + Vanilla JS** | 无框架依赖，直接可运行 |
| 部署 | 单进程 Flask 开发服务器 | 初期本地/小 VPS 运行即可 |

## 三、数据模型

### 3.1 数据库表结构（单表设计）

**表名：`questions`**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | TEXT (UUID) | 主键，唯一标识每个问题 |
| `content` | TEXT | 问题内容，上限 200 字 |
| `asker_cookie_id` | TEXT | 提问者的匿名 ID（存于浏览器 Cookie） |
| `asker_ip` | TEXT | 提问者 IP 地址（仅管理员可见） |
| `is_answered` | BOOLEAN | 是否已回答，默认 false |
| `answer_content` | TEXT | 回答内容（可为空） |
| `is_visible` | BOOLEAN | 是否在公开页面展示，默认 false |
| `created_at` | DATETIME | 提问时间 |
| `answered_at` | DATETIME | 回答时间（可为空） |

### 3.2 设计要点

- **单一匿名 ID**：`asker_cookie_id` 是一串随机生成的长字符串（如 UUID），在用户首次访问时写入浏览器 Cookie，不会过期。这串 ID 对所有人不可见，只存在于用户自己的浏览器中。
- **IP 仅管理员可见**：提问者 IP 只记录，不在任何公开页面展示。
- **双重控制**：`is_answered` 追踪是否已回答，`is_visible` 控制是否公开。两者独立，允许"已回答但不公开"的状态。

## 四、匿名追踪机制（核心设计）

这是本项目最关键的设计决策。以下是完整的匿名追踪方案：

### 4.1 Cookie 匿名 ID

```
用户首次访问 → 服务端生成随机 UUID → 写入 Cookie（httpOnly, 永不过期）
                      ↓
用户提问时 → 问题与该 Cookie ID 绑定
                      ↓
用户访问「我的问题」→ 用 Cookie ID 查询所有属于自己的问题
```

### 4.2 三种可见性模式

管理员在回答每个问题时，可以选择以下三种可见性：

| 模式 | `is_visible` | 行为 |
|------|:---:|------|
| **完全公开** | ✅ | 问答出现在公开浏览页，所有人都能看到 |
| **仅提问者可见** | ❌ | 问答只出现在提问者的「我的问题」页面 |
| **暂不展示** | ❌ | 已回答但暂不公开（与"仅提问者可见"共用 is_visible=false，通过是否已回答区分） |

> 简化版：实际上只需要 `is_visible` 一个开关。`is_visible=true` → 公开；`is_visible=false` + `is_answered=true` → 仅提问者可见。

### 4.3 「我的问题」页面（`/my`）

- 不需要登录，完全依赖 Cookie 中的匿名 ID。
- 展示该 Cookie 对应的所有问题及其回答状态：
  - **未回答**：显示"等待回答中…"
  - **已回答 + 公开**：显示问题和回答（也可引导去公开页查看）
  - **已回答 + 不公开**：显示问题和回答（仅此处可见）
- 用户换浏览器/清除 Cookie 后将**无法找回**之前的问题——这是匿名机制的取舍，需在页面上明确说明。

## 五、页面设计

### 5.1 公开提问 & 浏览页（`/`）

```
┌──────────────────────────────────┐
│  🔒 匿名提问箱                    │
│  有什么想问的？大胆说出来            │
│                                  │
│  ┌──────────────────────┐        │
│  │    ✏️ 我要提问        │ 按钮    │
│  └──────────────────────┘        │
│                                  │
│  ── 已公开的回答 ──────────────   │
│                                  │
│  ┌──────────────────────────┐    │
│  │ Q: 你今天吃了吗？      ▼  │    │ ← 折叠状态，点击展开
│  └──────────────────────────┘    │
│  ┌──────────────────────────┐    │
│  │ Q: 今天吃过了          ▼  │    │
│  │ A: 吃了火锅，很好吃       │    │ ← 展开状态
│  └──────────────────────────┘    │
│  ┌──────────────────────────┐    │
│  │ Q: 你喜欢什么颜色？   ▼  │    │

│                                  │
│  [ 我的问题 ] ← 链接到 /my       │
└──────────────────────────────────┘
```

**交互细节：**
- 点击「我要提问」→ 弹出模态框，内含 textarea（200 字上限 + 实时字数统计）+ 提交按钮。
- 提交成功后提示"提问成功！你可以在「我的问题」中查看回复状态"，并自动设置匿名 Cookie。
- 问答卡片默认折叠，只显示问题文本第一行（超出省略）。点击展开后显示完整问题和回答。
- 公开页面只展示 `is_visible=true` 且 `is_answered=true` 的问题。

**交互细节：**
- 进入 `/admin` 时要求输入密码（在服务端配置中设定，或通过环境变量）。
- 验证通过后设置 session，后续请求免密。
- 问题列表可筛选：全部 / 未回答 / 已回答 / 已公开。
- 每个问题卡片展示：IP 地址、问题内容、回答状态标签、展示状态标签。
- 点击展开后显示回答编辑区和操作按钮。
- 操作按钮：
  - **未回答的问题**：「保存回答」+ 「保存并展示」（一键完成回答+公开）
  - **已回答的问题**：「修改回答」+「展示」/「隐藏」（切换 is_visible）

## 六、API 设计

### 6.1 公开接口

| 方法 | 路径 | 说明 | 参数 |
|------|------|------|------|
| GET | `/api/questions/visible` | 获取所有公开展示的问答 | 无 |
| POST | `/api/questions` | 提交新问题 | `{ content: string }` |

### 6.2 匿名用户接口

| 方法 | 路径 | 说明 | 参数 |
|------|------|------|------|
| GET | `/api/my-questions` | 获取当前 Cookie 对应的所有问题 | 从 Cookie 读取 asker_id |

### 6.3 管理员接口（需 session 认证）

| 方法 | 路径 | 说明 | 参数 |
|------|------|------|------|
| POST | `/api/admin/login` | 管理员登录 | `{ password: string }` |
| GET | `/api/admin/questions` | 获取所有问题 | `?filter=all\|unanswered\|answered\|visible` |
| PUT | `/api/admin/questions/<id>` | 更新问题的回答和展示状态 | `{ answer_content?, is_visible? }` |

## 八、安全考量

| 风险 | 应对措施 |
|------|----------|
| 管理员密码泄露 | 密码通过环境变量传入，不写在代码中 |
| 恶意灌水 | 基于 IP 的频率限制（如每分钟最多 3 个问题） |
| XSS 攻击 | 所有用户输入在渲染时做 HTML 转义 |
| SQL 注入 | 使用参数化查询（SQLite + Flask 默认行为） |
| 管理员 session 劫持 | Session 使用 Flask 默认的 signed cookie |

## 九、用户流程总结

```
提问者视角：
  访问首页 → 点击「我要提问」→ 填写问题（≤200字）→ 提交
       ↓
  点击「我的问题」→ 看到自己的所有问题及回复状态
       ↓
  如果回答了且公开 → 可以在首页看到（和其他人一样）
  如果回答了但私密 → 只能在「我的问题」页面看到
  如果还没回答 → 显示等待中

浏览者视角：
  访问首页 → 浏览已公开的问答卡片 → 点击展开查看回答

管理员视角：
  访问 /admin → 输入密码 → 查看问题列表
       ↓
  筛选未回答的问题 → 展开 → 填写回答 → 保存
       ↓
  决定是否公开 → 点击「展示」/「隐藏」
```
